この文書は **base 約 327M** を実際に学習してみて残した記録です。
きれいなスコア表が目的ではなく、

- 学習はどうやったか
- データは何を使ったか
- 質問に何と答えたか

を残す方に寄っています。

### まず一行で

| | 結果 |
|:--|:-----|
| **Pretrain only** | チャットプロンプトにほぼ反応しない (28 問 0 点水準) |
| **SFT 後** | 指示形式は追おうとする。中身はよく間違う |
| **THINKING モード** | 回答が空になるバグ (14 問中 答 0 個) → 通常チャットモードで評価 |

まだ実使用の段階ではありません。
ただし **パイプラインが本当に回るか**、SFT が何を変えるかを見せる最初のスナップショットです。

## 1. 学習の過程

### モデル一枚まとめ

| 項目 | 値 |
|:-----|:---|
| 系統 | Decoder-only Transformer (LLaMA スタイル構成) |
| パラメータ | 約 **326.7M** (`base` プリセット) |
| 深さ · 幅 | 24 layer · d_model 1024 |
| Attention | GQA (Q 16 · KV 4) + RoPE |
| FFN | SwiGLU |
| 正規化 | RMSNorm (Pre-Norm) |
| その他 | weight tying、bias なし |

```text
入力トークン
   │
埋め込み ──────────────────────────────┐
   │                                 │ (weight tying)
[ Block × 24 ]                       │
  RMSNorm → Attention → +            │
  RMSNorm → SwiGLU    → +            │
   │                                 │
RMSNorm                              │
   │                                 │
LM Head ◄────────────────────────────┘
   │
次トークンの確率
```

### 学習の順序

```text
  ① BPE トークナイザー (vocab 64k)
           │
           ▼
  ② Pretrain · 18,000 step · lr 6e-4
           │
           ▼  pretrain_base_v1
  ③ SFT · 5,000 step · lr 3e-5
           │
           ▼  sft_base_v1
  ④ ベンチマーク (この文書)
```

| 段階 | チェックポイント | steps | 開始点 | データ指紋 |
|:-----|:-----------------|------:|:-------|:------------|
| Pretrain | `pretrain_base_v1` | 18,000 | ゼロから | `train.bin` · `53f815d47abc4887` |
| SFT | `sft_base_v1` | 5,000 | pretrain から続き | `sft.pt` · `bbc2211091309d3c` |

共通: AdamW (β 0.9 / 0.95, weight decay 0.1)、grad clip 1.0、warmup + cosine、CUDA。

### Loss がどう下がったか

**Pretrain**

| step | train | val |
|----:|------:|----:|
| 0 | 11.29 | — |
| 500 | 3.42 | 4.35 |
| 1,000 | 2.76 | 4.08 |
| 1,500 | 3.00 | 3.25 |
| 16,500 | | 2.72 |
| 17,500 | | 2.81 |
| 終盤付近 | 約 2.1 ~ 2.5 | |

**SFT** (pretrain チェックポイントから続き)

| step | train |
|----:|------:|
| 0 | 2.67 |
| 500 | 1.60 |
| 1,000 | 1.39 |
| 4,950 | **1.05** |

Loss が下がったのは「対話形式に適応中」という信号です。
ベンチの点数がすぐ良くなるという意味ではありません。下の Q&A を見ると、そのギャップがわかります。

## 4. スコア要約

### sft_base_v1

| モード | 結果 |
|:-----|:-----|
| THINKING **オン** | 答生成 **0 / 14** · 平均 0.0 |
| 通常チャット · 韓国語 | 平均 **1.33 / 5** |
| 通常チャット · 日本語 | 平均 **0.33 / 5** |
| 通常チャット · 英語 | 平均 **2.67 / 5** |
| 通常チャット · コーディング | テスト **4 / 17** · 問題完全通過 **1 / 5** |

THINKING モードでは閉じタグを書く前に生成が終わってしまいます。
推論コードはその場合答を空にするため、**このチェックポイントは通常チャットモードで使う方が正しいです。**

## 5. Pretrain と SFT の比較

| | Pretrain | SFT 後 |
|:--|:---------|:---------|
| 全体スコア | 事実上ほぼ全部 0 | 通常モードでスコア発生 |
| コーディングテスト | 0 / 34 | 4 / 17 (通常モード) |
| 振る舞い | 質問の書き写し、同じ言葉の繰り返し | 答の形式を合わせようとする (中身はよく間違う) |

SFT 5,000 step だけでも  
**「全く従えない → 形式は試す」** へ一段上がります。  
その上の品質はまだ遠いです。

## 7. コーディング設問

関数シグネチャだけ渡し、取り出したコードをテストで回しました。  
以下は **通常チャットモード** (THINKING モードのコーディングは全部失敗)。

| 問題 | テスト | スコア | 一行評 |
|:-----|:------:|:----:|:---------|
| is_prime | 0 / 5 | 0 | 偶数判定に変質 |
| reverse_string | 0 / 3 | 0 | lower だけで終わり |
| factorial | 0 / 3 | 1 | 再帰の骨格はあるが 0 処理なし |
| **is_palindrome** | **3 / 3** | **5** | **唯一きれいな正解** |
| find_max | 1 / 3 | 1 | 変数の上書き、運良く 1 ケース |

### うまくいった答 (is_palindrome)

```python
def is_palindrome(s):
    s = ''.join(e for e in s if e.isalnum())
    return s == s[::-1]
```

### 失敗例 (is_prime → 事実上偶数検査)

```python
def is_prime(n):
    if n <= 1:
        return False
    return n % 2 == 0
```

### 失敗例 (reverse_string)

```python
def reverse_string(s):
    return s.lower()
```
